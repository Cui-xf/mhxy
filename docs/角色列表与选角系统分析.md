## 一、网络协议

### 协议码 0x2003（8195）：服务端下发角色列表

### 角色列表数据结构（`GlobalStatus.a(DataInputStream)`）

```
byte    : 角色数量 N（为 0 则清空）
[N 条]
  UTF   : W[i]  角色 ID（服务端唯一标识）
  short : X[i]  等级
  byte  : Y[i]  性别（0=男, 1=女）
  byte  : Z[i]  种族（0=人族, 1=仙族, 2=妖族）
  UTF   : aa[i] 昵称（显示用）
  byte  : nD[i] 预留字节
[N 条，第二轮]
  UTF × 3        （跳过，暂未使用）
UTF              （全局预留字符串）
```

相关全局数组（均在 `GlobalStatus`）：
| 字段 | 类型 | 含义 |
|------|------|------|
| `W`  | `String[]` | 角色 ID |
| `X`  | `short[]`  | 等级 |
| `Y`  | `byte[]`   | 性别 |
| `Z`  | `byte[]`   | 种族 |
| `aa` | `String[]` | 昵称 |
| `nD` | `byte[]`   | 预留 |

## 三、角色精灵资源加载

### 精灵名称构建规则

资源名格式：`f{V[种族]}{W[性别]}{动作}{X[3]}{镜像}`

```java

    .append(var5)                  // 动作，通常=1
    .append(GlobalConfig.X[3])    // X[3]='1'，固定
    .append(mirror ? 1 : 2);       // 1=正向，2=镜像
```

## 四、交互逻辑

### 格子选择（触控）

| 触控方向 | 效果 |
|----------|------|
| 上/下 | selectActorRow 切换 0↔1 |
| 左/右 | selectActorClo 切换 0↔1 |

切换后更新底部文字控件 `at`，显示选中角色的昵称/等级。

### 按键/点击确认


### 创建角色（无角色时）

- 入口：`this.C()`
- 请求协议码：`0x1007`（4103）
- 创建页面用 `this.bx = new Frame1[6]` 加载 6 种职业预览动画

### 删除角色

- 弹出确认框（输入"OK"）
- 确认后发送删除请求，服务端回包后重新调用 `b(charCount)`

## 五、已知 Bug 记录

### Bug：角色头像/动画不显示

**文件**：`MainCanvas.java:1569`

**原因**：资源名前缀多了 `"com.cc."`，导致 hashKey 不匹配，Frame1 加载返回 null。

```java
// 错误（修复前）
GlobalConfig.ag.append("com.cc.f").append(...)
// → hashKey("com.cc.f10111s") = -1845103537，rule 中不存在

// 正确（修复后）
GlobalConfig.ag.append("f").append(...)
// → hashKey("f10111s") = 1779728135，rule 中 id=235
```
