# t_1 类分析文档

## 概述

`t_1` 是《梦幻西游》手游客户端（J2ME/MIDlet 平台）的**全局配置与工具类**，以静态字段和静态方法的形式集中管理游戏运行时的各类全局状态、常量数据及通用工具函数。

## 方法说明

### `a(MIDlet midlet)` — 初始化方法

从 JAD（Java Application Descriptor）属性中读取运行时配置：

- 读取 `PopularizeChannel` 属性，根据渠道号设置对应的渠道特性布尔标志（`x`~`D`）
- 读取 `hasRideRes` 或 `LargeClient` 属性，决定是否加载坐骑资源
- 读取 `Model` 属性获取设备型号
- 若 `a==0`（内部包），根据 `r` 的值设置渠道字符串 `v`

### `a(int width, String text)` — 文本水平居中偏移计算

```java
public static int a(int var0, String var1)
```
返回将字符串在给定宽度内水平居中所需的 x 偏移量。

### `a(int width)` — 字符居中偏移计算

```java
public static int a(int var0)
```
返回单个字符（宽度为 `j`）在给定宽度内水平居中的 x 偏移量。

### `a(int, byte, byte, int, boolean)` — 构建资源路径字符串

```java
public static String a(int var0, byte var1, byte var2, int var3, boolean var4)
```
利用 `V`、`W`、`X` 字符数组和 `ag` 缓冲区拼接资源标识字符串，格式如：`{V[var0]}{W[var1]}{var3}{X[3]}`。当 `m==2` 时强制 `var3=0`。

### `a(String)` — 空方法（占位）

```java

```
方法体为空，可能为调试占位或已被移除的功能。

### `a(StringBuffer)` — 清空 StringBuffer

```java

```
将传入的 `StringBuffer` 清空（`delete(0, length())`）。

### `a(byte, byte)` — 编码转短整型

```java

```
将 `V[var0]` 和 `W[var1]` 拼接为字符串后解析为 `short`，用于编码转换。

### `a(StringBuffer, long)` — 银两格式化

```java
public static String a(StringBuffer var0, long var1)
```
将长整型银两数值格式化为中文单位字符串，规则如下：
- `亿`单位：`÷ 100,000,000`
- `万`单位：余数 `÷ 10,000`
- 个位：最终余数
- 全0时显示 `"0两"`
- 结尾统一追加 `"两"`

示例：`123456789` → `"1亿2345万6789两"`

## 静态初始化块（`static {}`）

在类加载时执行，完成所有静态字段的初始值设置：

1. 初始化字体对象（`h`、`i`），计算字体高度和汉字宽度
2. 设置客户端控制标志的默认值
3. 填充所有 UI 文本字符串数组（频道、门派、功能、装备等）
4. 初始化战斗坐标数组 `S`、`T`
5. 初始化字符映射数组 `V`、`W`、`X`
6. 初始化全局 `StringBuffer ag`

## 总结

`t_1` 类是整个客户端的**配置中枢**，承担以下职责：

1. **全局常量存储**：集中管理所有 UI 文本、游戏规则字符串、坐标数据
2. **渠道差异化控制**：根据 JAD 中的渠道号动态启用/禁用特定功能
3. **通用工具函数**：提供字符串居中、银两格式化、资源路径构建等复用工具
4. **性能优化**：通过复用 `ag`（StringBuffer）减少对象分配，适应 J2ME 内存受限环境
5. **运行时初始化入口**：`a(MIDlet)` 方法在游戏启动时被调用，完成运行时参数注入

该类的命名（单字母字段和方法）是典型的 J2ME 代码混淆（obfuscation）特征，原始源码可读性更高。
