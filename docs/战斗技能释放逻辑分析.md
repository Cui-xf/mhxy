# 战斗技能释放逻辑分析

> 基于 `mhxy_s` Java 项目源码逆向分析，代码经过反混淆还原，部分类名/字段名保留了原始短名

---

## 一、整体架构概览

战斗技能释放涉及以下几个核心模块：

| 模块 | 类 | 职责 |
|------|-----|------|
| 战斗控制 | `FightModel` | 战斗状态机、用户输入处理、回合驱动 |
| 战斗角色数据 | `FightRoleData` | 单个战斗单元的状态与动画绑定 |
| 技能动画 | `SkillAnim` | 技能特效动画的实例（坐标 + Animation 引用） |
| 资源管理 | `ResourceManager` | 动画/精灵资源的加载与缓存 |
| 动画数据 | `Animation` | 多帧动画对象（帧列表、播放状态） |
| 精灵片段 | `SpritePiece` | 单帧中的一个绘制单元 |
| 渲染工具 | `PngUtil` | 动画帧推进（`animate`）与绘制 |
| 网络包处理 | `NetworkPacketProcessors` | 解析服务器下发的战斗结果包（8206） |

## 二、资源加载流程

### 2.1 ResourceManager 的三类资源

```
type=0  → "名称" + "p"  → Sprite（静态图集）
type=2  → "名称" + "s"  → Animation（帧动画，用于角色/技能）
type=3  → "名称" + "m"  → TileMap（地图对象）
```

### 2.2 战斗相关 ResourceManager 实例

在 `MainCanvas` 中存在多个全局 `ResourceManager` 实例，按角色类型分类：

- `MainCanvas.role`     — 人物角色动画（文件名规则：`f{id}0{dir}{action}`）
- `MainCanvas.petfight` — 宠物战斗动画（文件名规则：`{id}{action}`，支持 HSL 染色变体）
- `MainCanvas.skill`    — 技能特效动画（文件名规则：直接使用技能 ID 数字字符串）

### 2.3 动画加载核心流程

```
loadAnimation(path, h, s, l)
  └─ buildResourceId(path, type=2)      // "path" + "s" 后做 hashKey
  └─ 命中缓存 → 直接返回
  └─ 未命中 → getFrameInfoByKey()       // 在 frameInfos[] 索引表中查找
             → getFrameStream()         // 从 JAR/文件系统读取字节流
             → loadAnimationFromFile2Cache()
                 ├─ parseSpriteData()   // 读帧数据，构建 Animation 对象
                 ├─ 收集所有 SpritePiece.spriteId
                 ├─ loadSpriteToCache() // 批量加载依赖的 Sprite 图集
                 └─ frameCache.addElement(animation)
```

**关键细节：**
- `frameInfos[]` 在构造时从索引文件（`frameInfo` 字节流）读入，每条记录包含 `(int key, short id)`
- 资源名的哈希算法：`hash = hash * 31 + char`（Java `String.hashCode` 同款多项式）
- HSL 染色版本（宠物换色）通过 `copy(h,s,l)` 深拷贝 `SpritePiece` 数组后单独缓存，共享底层 `Sprite` 图集

### 2.4 Animation.build() 二进制格式

```
byte  num        // 共几段（帧数）
if num > 0:
  short x, y, w, h   // 整体包围盒
  for i in 0..num:
    byte frameNum      // 本段有几个 SpritePiece
    for j in 0..frameNum:
      SpritePiece.build(dis)
        short spriteId
        byte  type
        short transformX, transformY
        short imageSliceIndex
        byte  transformFlag
  short frameInterval  // 帧间隔（毫秒）
```

## 三、FightRoleData — 战斗角色动画绑定

`FightRoleData` 描述一个战斗单元（我方角色/NPC/宠物），关键字段：

| 字段 | 含义 |
|------|------|
| `a` (byte) | 战场位置编号（0-5） |
| `b` (byte) | 角色类型：0=人物，1=宠物（敌方），2=宠物（己方） |
| `e` (int) | 当前 HP（≤0 表示死亡） |
| `i` (short) | 外观资源 ID |
| `j` (Animation) | 当前播放的动画对象 |
| `k` (byte) | 动画状态：1=待机，2=攻击/施法，4=受击/死亡 |
| `p/q/r` (short) | HSL 染色参数（宠物换色） |

### 3.1 动画绑定方法 b(byte action)

根据角色类型拼接动画名并从缓存中取出 `Animation`：

```java
// 人物 (b==0)
// action != 4 → "f" + id + "0" + dir + action
// action == 4 → "f" + id + "0" + action   （死亡动画无方向）
animName = "f" + this.i + "0" + this.t + var1;
this.j = MainCanvas.role.getAnimationByNameFromCache(animName);
// 若为空，尝试 dir=1 的默认朝向

// 宠物 (b==1 or b==2)
// action > 3 时截断为 3
animName = String.valueOf(this.i) + (var1 > 3 ? 3 : var1);
this.j = MainCanvas.petfight.getAnimationByNameFromCache(animName, p, q, r);
// 若为空，尝试 id=100 的默认宠物
```

**动作编号含义（推断）：**

| action 值 | 含义 |
|-----------|------|
| 1 | 待机/站立 |
| 2 | 攻击/施法 |
| 3 | 移动/跳跃 |
| 4 | 死亡 |

## 四、SkillAnim — 技能特效动画

`SkillAnim` 用于在战场上播放覆盖式技能特效，独立于角色动画：

```java
public class SkillAnim {
    public Animation animation;  // 技能动画资源（从 MainCanvas.skill 缓存取）
    public int[] framW;          // 每个受击目标的屏幕 X 坐标
    public int[] framH;          // 每个受击目标的屏幕 Y 坐标
}
```

**构造逻辑（`SkillAnim(int skillId, int targetCount)`）：**

1. 从 `MainCanvas.skill.getAnimationByNameFromCache(String.valueOf(skillId))` 获取动画
2. 若动画数据无效，回退到 `skillId=10` 的默认技能动画
3. 根据受击目标数量分配坐标数组

**坐标来源：**
- 敌方目标 → `GlobalConfig.R[seat][0/1]`（敌方战场坐标表）
- 我方目标 → `GlobalConfig.Q[seat][0/1]`（我方战场坐标表）
- `seat` = `ae.c`（受击目标的位置编号，0-5）

## 五、技能释放完整流程

### 5.1 用户操作阶段（FightModel 状态机）

`FightModel` 内部使用字段 `d`（操作阶段）和 `e`（菜单光标位置）驱动战斗 UI 状态机：

```
d=0 → 主菜单（攻击/防御/技能/道具/逃跑）
d=1 → 选择攻击目标（针对敌方）
d=2 → 选择宠物技能目标（针对敌方）
d=3/4/5 → 指定目标后的二次确认
```

**技能释放选择路径（以主动技能为例）：**

```
用户在菜单选 e=2（技能）
  → 打开技能列表（gameSceneController.h(3)）
  → 选择技能 → 设置 j=1, p=skillId
  → d=2（进入选目标阶段）
  → 确认目标 → 调用 a(j, p, n, o, k, q, l, targetId)
              → 发送网络包 4180
              → f = -1（等待服务器回包）
```

### 5.2 发送攻击包（4180）

```java
// FightModel.a(byte actionType, int skillId, byte n, byte targetId, ...)
NetPayloadBuilder.a((short)4180,
    GlobalStatus.roleId_2,  // 角色ID
    GlobalStatus.v,         // fightId（战斗会话ID）
    g,                      // 全局战斗序号
    t,                      // 战斗类型（0=PVE, 1/2=PVP）
    actionType,             // 0=物品, 1=技能, 2=装备, 3=宠物技能, 4=逃跑
    skillId,                // 技能/道具ID
    n,                      // 目标阵营（0=对方, 1=己方）
    targetId,               // 目标位置编号
    ...
);
```

### 5.3 接收服务器战斗结果包（8206）

`NetworkPacketProcessors.b(DataInputStream)` 解析 8206 包：

```
long fightId         // 战斗会话ID（与本地 GlobalStatus.v 对比校验）
short roundCount     // 本次有几个回合的结算
for each round:
  short actionCount  // 本回合有几条行动记录
  for each action:   // → 构造 i[] 数组
    byte a           // 施法者阵营（0=我方, 1=敌方）
    byte b           // 施法者位置
    byte  (skip)
    byte c           // 受击者阵营
    byte d           // 受击者位置
    short e          // 技能/道具 ID
    String f         // 文字描述（如"造成XX点伤害"）
  for each action:   // → 构造 ae[] 数组（伤害结算数据）
    byte a           // 施法者阵营
    byte b           // 施法者位置
    byte c           // 受击者位置
    int  d           // 技能 ID / 动作类型
    int  g           // 新 HP
    int  i           // 新最大HP
    int  e           // 伤害量/回复量
    int  h           // 新 MP
    int  j           // 新最大MP
    byte k           // 特效类型
  String text        // 回合文字消息
→ 数据推入 FightModel 的 v/w/x 队列
→ overlayDialogController.f = 1（触发战斗动画播放）
```

### 5.4 战斗动画播放阶段（FightModel.a(int) 帧更新）

`FightModel.a(int)` 在每帧被调用，`f` 字段表示当前动画状态：

```
f=0  → 等待用户操作（显示战斗菜单）
f=-1 → 等待服务器回包
f=1  → 从队列取出本回合数据，解析施法者/受击者，触发动画
f=2  → 播放施法者攻击动画（k=2），等待动画播放完毕
f=3  → 播放受击/结算（k=4），显示伤害数字
f=4  → 过渡状态
f=5  → 播放技能特效（skillAnim）
f=6  → 特殊状态（如被封印/混乱）
f=9  → 等待队友操作
```

### 5.5 技能特效（skillAnim）创建时机

在 `FightModel.o()` 中，服务器包中的 `ae.d`（动作类型）决定是否创建技能特效：

```java
// ae.d 含义（推断）：
// 1  → 普通攻击
// 2  → 被封印
// 3  → 被混乱
// 5  → 对敌方施法（创建 skillAnim 覆盖敌方位置）
// 8  → 对己方施法（如治疗，创建 skillAnim 覆盖我方位置）
// 10 → 对单一目标施法
// -1 → 直接进入结算

if (ae.d == 5 && ae.e >= 10) {
    skillAnim = new SkillAnim(ae.e /*技能ID*/, v() /*受击目标数*/);
    // 为每个受击目标分配坐标（GlobalConfig.R / GlobalConfig.Q）
}
// 创建完毕后：
fightData[target].b((byte) 2);  // 施法者切换到攻击动画
f = 2;                           // 进入动画播放阶段
```

**技能 ID 阈值（`>= 10`）：** 只有 ID >= 10 的技能才会创建 `SkillAnim` 特效，ID < 10 的（普通攻击等）只播放角色攻击动画。

### 5.6 动画播放阶段状态转换

```
f=2（施法者攻击动画）
  → FightRoleData.k = 2，播放攻击动作
  → PngUtil.animate() 返回 2（一轮播完）
  → 若有 skillAnim → f=5（播技能特效）
  → 若无 skillAnim → 受击者切换到 k=4 → f=3（结算）

f=5（技能特效）
  → skillAnim.a(graphics, pngUtil, 0, 0)
    └─ PngUtil.roleSelectedAnimation() 在各目标坐标绘制 animation 当前帧
  → s() 检测 PngUtil.animate(skillAnim.animation) == 2
  → 特效播完 → 受击者切换到 k=4 → f=3（结算）

f=3（受击/结算）
  → 显示伤害数字浮动（ae.d / ae.e 数值）
  → B 时间戳到期后 → f=0 或进入下一回合
```

## 六、动画渲染流程

### 6.1 PngUtil.animate()：帧推进

```java
public static int animate(Animation frame1, long frameStartTs) {
    if (frameStartTs - frame1.lastTs - frame1.frameInterval > 0) {
        frame1.currentFrameIndex++;
        if (frame1.currentFrameIndex >= frame1.spritePieces.length) {
            frame1.currentFrameIndex = 0;
            return 2;  // 一轮播完（循环重置）
        }
        frame1.lastTs = frameStartTs;
        return 1;      // 正常推进
    }
    return 0;          // 时间未到，保持当前帧
}
```

### 6.2 PngUtil.roleSelectedAnimation()：绘制一帧

```java
// 取当前帧的所有 SpritePiece，逐一绘制
for (SpritePiece piece : animation.spritePieces[currentFrameIndex]) {
    switch (piece.frame.type) {
        case 0: // Sprite 静态图
            draw(graphics, (Sprite)piece.frame,
                 clip, piece.getImageSlice(),
                 x, y,
                 w + piece.transformX, h + piece.transformY,
                 20/*TOP_LEFT*/, piece.transformFlag);
            break;
        case 2: // 嵌套 Animation
            roleSelectedAnimation(graphics, (Animation)piece.frame,
                                  clip, x, y,
                                  w + piece.transformX, h + piece.transformY,
                                  20, ignore);
            break;
    }
}
```

### 6.3 坐标系说明

战场坐标预先计算并存储在 `GlobalConfig`：

```
GlobalConfig.R[seat][0/1]  → 敌方 seat 位的屏幕坐标 (x, y)
GlobalConfig.Q[seat][0/1]  → 我方 seat 位的屏幕坐标 (x, y)
seat ∈ [0, 5]，共 6 个位置（前3后3）
```

坐标基准来自 `GlobalConfig.ZhanDouZuoBiao1/2`，在 `FightModel.a()` 初始化时根据屏幕分辨率动态偏移调整。

## 七、数据流汇总

```
用户点击技能
    │
    ▼
FightModel.a(int key)           // 按键处理
    │  设置 j=1(技能), p=skillId, n/o=目标
    │  调用 FightModel.a(byte,int,byte,byte,...)
    │
    ▼
NetPayloadBuilder.a(4180, ...)  // 组包
    │
    ▼
MainCanvas.netUtils.sendPacket  // 发送
    │
    │  [网络等待]
    ▼
NetworkPacketProcessors.b()     // 收到 8206
    │  解析 i[]（行动描述）+ ae[]（数值结算）
    │  推入 FightModel.v / w 队列
    │  设置 overlayDialogController.f = 1
    │
    ▼
FightModel 帧循环 (f=1)
    │  从队列取 E(i数组) / F(ae数组)
    │  调用 o() 确定施法/受击角色
    │  FightRoleData.b(2)  → 施法者切换攻击动画
    │  若 skill >= 10 → new SkillAnim(skillId, targetCount)
    │  f = 2
    │
    ▼
FightModel 帧循环 (f=2)
    │  PngUtil.animate(施法者.j)
    │  动画播完 → 有 skillAnim → f=5
    │              无 skillAnim → 受击者.b(4) → f=3
    │
    ▼
FightModel 帧循环 (f=5)         // 技能特效
    │  skillAnim.a(graphics,pngUtil,0,0)
    │    └─ PngUtil.roleSelectedAnimation(skillAnim.animation, 坐标)
    │  PngUtil.animate(skillAnim.animation) == 2
    │  特效播完 → 受击者.b(4) → f=3
    │
    ▼
FightModel 帧循环 (f=3)         // 受击/结算
    │  受击者播放受击动画 k=4
    │  绘制伤害数字浮动
    │  FightRoleData.e/g 更新 HP/MP
    │  → f=0（回合结束，等待下一操作）
```

## 九、技能资源命名规则总结

| 资源类型 | 来源方法 | 原始名规则 | 示例 |
|----------|----------|-----------|------|
| 战斗/展示人物动画 | `FightRoleData.b()` | `f{appearId}{mid}{dir_char}{action}` | `f30011`、`f31111` |
| 场景移动动画（无坐骑） | `buildRoleResStr` | `{jobChar}{genderChar}{appearance}{dirChar}` | `3100`、`3101` |
| 场景攻击动画（无坐骑） | `buildRoleResStr` | `{jobChar}{genderChar}{appearance}{dirChar}4` | `31004`、`31014` |
| 宠物战斗 | `FightRoleData.b()` | `{宠物ID}{动作}` | `1001`、`1002` |
| 技能特效 | `FightModel.o()` | `{技能ID}` | `22`、`14` |

> 注意：两套规则中末尾 `4` 含义不同：`buildRoleResStr` 里 `4` = **攻击**（isAttack=true）；`FightRoleData` 里 `4` = **死亡**动作。

- Java 侧后缀 `s` 自动附加（`wrapName`），即实际查找 hash 时用 `f30011s`
- 技能 ID < 10 不创建 `SkillAnim`（普通攻击等），直接复用角色攻击动画

### 人物动画（role，`f` 前缀系列，来自 `FightRoleData`）

**拼接规则：** `f{appearId}{mid}{dir_char}{action}`

| 字段 | 取值 | 说明 |
|------|------|------|
| `appearId` | `10`/`11`/`20`/`21`/`30`/`31` | 两位外观ID = `roleJobResIds[job]` + `roleGenderResIds[gender]` |
| `mid` | `0` / `1` | `0`=战斗侧面（`FightRoleData`），`1`=正面展示（角色选择页等） |
| `dir_char` | `0` / `1` | 方向字符（`this.t`）；死亡时无此字段 |
| `action` | `1`=待机，`2`=攻击，`4`=死亡 | 死亡格式：`f{appearId}{mid}4`（无 dir_char） |

> 资源库中仅有 `dir_char=1` 的动画；`dir_char=0` 及死亡动画均不存在于资源包中。

**资源库中实际存在的文件：**

| appearId | 侧面待机 `f{id}011` | 侧面攻击 `f{id}012` | 正面待机 `f{id}111` | 正面攻击 `f{id}112` |
|----------|--------------------|--------------------|--------------------|--------------------|
| 10 (job=1,gender=0) | `f10011.anim` | `f10012.anim` | `f10111.anim` | `f10112.anim` |
| 11 (job=1,gender=1) | `f11011.anim` | `f11012.anim` | `f11111.anim` | `f11112.anim` |
| 20 (job=2,gender=0) | `f20011.anim` | `f20012.anim` | `f20111.anim` | `f20112.anim` |
| 21 (job=2,gender=1) | `f21011.anim` | `f21012.anim` | `f21111.anim` | `f21112.anim` |
| 30 (job=3,gender=0) | `f30011.anim` | `f30012.anim` | `f30111.anim` | `f30112.anim` |
| 31 (job=3,gender=1) | `f31011.anim` | `f31012.anim` | `f31111.anim` | `f31112.anim` |

### 场景移动/攻击动画（role，无前缀，来自 `buildRoleResStr`）

**拼接规则：**
- 移动：`{jobChar}{genderChar}{appearance}{dirChar}`
- 攻击：`{jobChar}{genderChar}{appearance}{dirChar}4`（末尾 `4` = isAttack=true）

| 字段 | 取值 | 说明 |
|------|------|------|
| `jobChar` | `1`/`2`/`3` | `roleJobResIds[job]` |
| `genderChar` | `0`/`1` | `roleGenderResIds[gender]` |
| `appearance` | `0`..`N` | 时装外观ID，`0`=默认无时装 |
| `dirChar` | `0`/`1`/`4` | `roleDirectionResId[direction]`，dir=0,1→`0`，dir=2,3→`1`，dir=4→`4` |

> 资源库中仅 appearance=0 的动画存在，时装外观动画不在资源包中。

**资源库中实际存在的文件（appearance=0）：**

| 移动(dir→0) | 移动(dir→1) | 攻击(dir→0) | 攻击(dir→1) |
